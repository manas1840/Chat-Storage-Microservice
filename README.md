# RAG Chat Storage Microservice

A production-ready Spring Boot microservice for storing and managing chat histories generated by a RAG (Retrieval-Augmented Generation) chatbot system.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Language | Java 21 |
| Framework | Spring Boot 3.3.0 |
| Security | Spring Security (Basic Auth) |
| Database | H2 (in-memory) |
| ORM | Spring Data JPA / Hibernate |
| Rate Limiting | Bucket4j |
| API Docs | SpringDoc OpenAPI / Swagger UI |
| Health | Spring Actuator |
| Testing | JUnit 5 + Mockito |
| Build | Maven |
| Container | Docker + Docker Compose |

---

## Features

- Create, rename, delete, and list chat sessions per user
- Mark / unmark sessions as favorites
- Add messages (USER / ASSISTANT / SYSTEM) with optional RAG context
- Paginated message retrieval
- Basic Authentication (stateless)
- Per-IP rate limiting via Bucket4j
- CORS configuration
- Centralized global error handling
- Structured logging with file output
- H2 in-memory DB with browser console
- Swagger UI for interactive API docs
- Health and info actuator endpoints
- Docker & Docker Compose support
- Unit tests for all service methods

---

## Quick Start (Local)

### 1. Clone the repository

```bash
git clone <repo-url>
cd rag-chat-service
```

### 2. Configure environment

```bash
cp .env.example .env
# Edit .env as needed
```

### 3. Run with Maven

```bash
mvn spring-boot:run
```

The service starts at **http://localhost:8080**

## Configuration

All settings are driven by environment variables. See `.env.example` for the full list.

| Variable | Default | Description |
|---|---|---|
| `SERVER_PORT` | `8080` | Port the service listens on |
| `DB_USERNAME` | `sa` | H2 database username |
| `DB_PASSWORD` | _(empty)_ | H2 database password |
| `APP_USERNAME` | `admin` | Basic auth username |
| `APP_PASSWORD` | `admin123` | Basic auth password |
| `RATE_LIMIT_CAPACITY` | `100` | Max tokens per bucket |
| `RATE_LIMIT_REFILL_TOKENS` | `100` | Tokens refilled per interval |
| `RATE_LIMIT_REFILL_SECONDS` | `60` | Refill interval in seconds |
| `CORS_ALLOWED_ORIGINS` | `http://localhost:3000,...` | Allowed CORS origins (comma-separated) |
| `DEFAULT_PAGE_SIZE` | `20` | Default page size for pagination |
| `LOG_FILE` | `logs/rag-chat-service.log` | Log file path |

## Authentication

All `/api/**` endpoints require **HTTP Basic Authentication**.

```bash
# Example
curl -u admin:admin123 http://localhost:8080/api/v1/sessions
```

In Swagger UI, click **Authorize** and enter your username/password.

## API Reference

Base path: `/api/v1`

### Chat Sessions

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/sessions` | Create a new chat session |
| `GET` | `/sessions` | List sessions (paginated, optional favorites filter) |
| `GET` | `/sessions/{id}` | Get a specific session |
| `PATCH` | `/sessions/{id}/rename` | Rename a session |
| `PATCH` | `/sessions/{id}/favorite` | Toggle favorite status |
| `DELETE` | `/sessions/{id}` | Delete a session and all its messages |

### Chat Messages

| Method | Endpoint | Description |
|---|---|---|
| `POST` | `/sessions/{id}/messages` | Add a message to a session |
| `GET` | `/sessions/{id}/messages` | Get messages (paginated) |

### System

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/actuator/health` | Health check (public) |
| `GET` | `/actuator/info` | App info (public) |
| `GET` | `/swagger-ui.html` | Swagger UI (public) |
| `GET` | `/api-docs` | OpenAPI JSON (public) |
| `GET` | `/h2-console` | H2 Browser Console (public, dev only) |

## Example Requests

### Create a session
```bash
curl -u admin:admin123 -X POST http://localhost:8080/api/v1/sessions \
  -H "Content-Type: application/json" \
  -d '{"name": "My first chat"}'
```

### Add a message
```bash
curl -u admin:admin123 -X POST http://localhost:8080/api/v1/sessions/1/messages \
  -H "Content-Type: application/json" \
  -d '{
    "sender": "USER",
    "content": "What is RAG?",
    "context": "Retrieved chunk: RAG stands for Retrieval-Augmented Generation..."
  }'
```

### List sessions with pagination
```bash
curl -u admin:admin123 "http://localhost:8080/api/v1/sessions?page=0&size=10&favoritesOnly=false"
```

### Rename a session
```bash
curl -u admin:admin123 -X PATCH http://localhost:8080/api/v1/sessions/1/rename \
  -H "Content-Type: application/json" \
  -d '{"name": "RAG Questions"}'
```
